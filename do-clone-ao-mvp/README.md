# Projeto MVP+ — Editor Markdown Inteligente

> Experiência 05 — Do Clone ao Produto Mínimo Viável (MVP+)

---

# 📌 Visão Geral

Este projeto consiste em um editor Markdown moderno desenvolvido com foco em produtividade, personalização visual e sincronização em nuvem.

O objetivo foi transformar um simples clone funcional em um **Produto Mínimo Viável aprimorado (MVP+)**, adicionando recursos que elevam a experiência do usuário e aproximam o sistema de uma aplicação real de mercado.

Além das funcionalidades básicas de edição, o projeto recebeu melhorias voltadas para:

- experiência visual;
- persistência de dados;
- usabilidade;
- integração com serviços externos;
- personalização do ambiente.

---

# 🧠 Questão A — Desenvolvimento Assistido por IA e o impacto no engenheiro júnior

A tendência de Desenvolvimento Assistido por IA está transformando o papel do engenheiro de software júnior. Ferramentas como Google Gemini, GitHub Copilot e ChatGPT automatizam tarefas que antes eram consideradas atividades típicas de entrada na carreira, como geração de código simples, documentação básica e correção inicial de bugs.

Com isso, o mercado deixa de valorizar apenas quem “sabe programar” e passa a exigir profissionais capazes de pensar criticamente sobre o software produzido pela IA.

## Competências indispensáveis

### 1. Capacidade de validação e pensamento crítico técnico

O engenheiro júnior precisará analisar se o código gerado pela IA é:

- seguro;
- eficiente;
- escalável;
- adequado ao problema proposto.

Embora a IA acelere o desenvolvimento, ela ainda pode gerar:

- erros lógicos;
- vulnerabilidades;
- código redundante;
- más práticas de arquitetura.

Por isso, conhecimentos sólidos em:

- algoritmos;
- arquitetura de software;
- engenharia de software;
- boas práticas de desenvolvimento;

continuam essenciais.

---

### 2. Habilidade de comunicação e formulação de prompts

Outra competência importante é saber interagir corretamente com a IA.

O profissional que consegue:

- descrever requisitos claramente;
- dividir problemas complexos;
- iterar instruções de forma objetiva;

obtém resultados muito superiores.

Nesse cenário, o engenheiro deixa de ser apenas um programador operacional e passa a atuar como um **orquestrador de soluções**.

---

### 3. Adaptabilidade e aprendizado contínuo

Como as ferramentas evoluem rapidamente, torna-se indispensável manter uma postura de aprendizado contínuo e adaptação constante às novas tecnologias.

# ⚖️ Questão B — Originalidade vs. Plágio Digital

## 1. Quando a engenharia reversa assistida por IA vira plágio digital?

A engenharia reversa pode ser legítima quando utilizada para:

- aprendizado;
- interoperabilidade;
- análise de segurança;
- inspiração conceitual.

O problema surge quando a IA é usada para reproduzir excessivamente:

- estrutura;
- identidade visual;
- lógica proprietária;
- experiência de uso;

de outro software sem autorização.

Ela passa a ser considerada plágio digital quando:

- existe cópia substancial de design ou funcionalidades;
- o objetivo é replicação comercial;
- a autoria intelectual original é comprometida;
- há tentativa de ocultar a origem da inspiração usando IA.

Portanto, o limite ético está menos na ferramenta e mais na intenção e no grau de reprodução do conteúdo original.

---

## 2. Diretriz ética para proteger inovação sem frear IA generativa

Uma abordagem viável seria um modelo de **transparência assistida por IA**.

Nesse modelo:

- desenvolvedores registrariam o uso relevante de IA;
- empresas utilizariam rastreabilidade de código e design;
- ferramentas generativas detectariam reproduções excessivas;
- equipes adotariam políticas de adaptação criativa.

Essa abordagem preserva a inovação sem impedir o uso produtivo da IA como ferramenta de apoio técnico.

# 🚀 Funcionalidades Implementadas

## 🎨 Dark Mode Minimalista

Todo o design foi reformulado para uma estética minimalista, moderna e focada na leitura e produtividade.

---

## 🌈 Sistema de Paletas (Accent Sync)

O editor agora possui suporte a múltiplos temas visuais:

- **Light Modern**
- **Default Dark**
- **Midnight Blue**
- **Deep Forest**
- **Crimson Red**

Cada paleta altera automaticamente:

- cores principais;
- destaque visual;
- fundo;
- componentes da interface.

---

## ☁️ Autenticação e Cloud Sync (Firebase)

Integração completa com Firebase.

### Recursos adicionados:

- Login com Google;
- Cloud Firestore;
- sincronização em tempo real;
- salvamento automático de preferências.

Os textos e temas do usuário ficam armazenados na nuvem.

---

## 📂 Drag-and-Drop Inteligente

Agora é possível:

- arrastar arquivos `.txt`;
- arrastar arquivos `.md`;
- inserir imagens diretamente no editor.

Tudo é carregado instantaneamente.

---

## 🔗 Compartilhamento Ágil

Foi adicionado um sistema rápido de compartilhamento que:

- copia automaticamente o Markdown atual;
- fornece feedback visual imediato ao usuário.

---

## ✍️ Auto-correção Gramatical

O editor utiliza o sistema nativo do navegador para correção gramatical automática, respeitando o idioma configurado no sistema operacional do usuário.
